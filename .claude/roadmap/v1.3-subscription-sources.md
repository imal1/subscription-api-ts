# v1.3 — 订阅源管理

**状态：待开发**

## 版本目标

支持多种订阅源输入方式，不再仅依赖 sing-box 本地配置，让服务成为一个通用的订阅聚合和转换平台。

## 核心功能

### 1. 多源订阅管理

**订阅源类型**：

| 类型 | 说明 | 示例 |
|------|------|------|
| Sing-box 配置 | 现有方式，通过 `sing-box url <name>` 获取 | `vless`, `hysteria2` |
| 远程 URL | 直接拉取远程订阅链接 | `https://example.com/sub?token=xxx` |
| 手动输入 | 在仪表盘直接粘贴节点 URL | 文本框批量粘贴 |
| 文件上传 | 上传 YAML/JSON/TXT 节点文件 | 拖拽上传 |

### 2. 订阅源管理页面

新增 `/sources` 路由：

- **源列表**：显示所有订阅源（名称、类型、状态、上次更新时间、节点数）
- **添加源**：选择类型 → 填写信息 → 测试连接 → 保存
- **编辑源**：修改源配置
- **删除源**：确认后删除
- **手动刷新**：单个源或全部源触发更新
- **源健康检查**：定期检测源可用性

### 3. 节点聚合

- **去重**：相同 `server:port` 的节点自动去重（可配置策略：保留第一个 / 保留最新的 / 全部保留）
- **节点过滤**：支持正则表达式过滤节点名称
- **节点排序**：按名称 / 协议 / 服务器 排序
- **节点重命名**：批量重命名规则（如添加前缀、替换关键字）

### 4. 订阅源健康监控

- **可用性检测**：每小时检测一次源是否可访问
- **延迟记录**：记录每次获取源的响应时间
- **失败通知**：源连续失败 3 次后在仪表盘显示警告
- **自动禁用**：源连续失败 10 次后自动禁用，避免拖慢整体更新
