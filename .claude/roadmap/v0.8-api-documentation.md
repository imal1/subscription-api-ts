# v0.8 — OpenAPI 文档 & API 完善

**状态：待开发**

## 版本目标

提供标准化的 API 文档，完善 API 的错误处理和响应格式，使服务可被第三方工具集成。

## 核心功能

### 1. OpenAPI 3.0 文档

- **规范文件**：生成 `openapi.yaml`，覆盖所有 API 端点
- **文档页面**：新增 `/api-docs` 路由（已有框架，需完善），使用 Swagger UI 或 Scalar 渲染
- **自动生成**：基于 TypeScript 类型和 JSDoc 注释生成 OpenAPI 规范（可选工具：`zod-to-openapi` 或手写）

### 2. API 响应标准化

- **统一错误格式**：

```json
{
  "success": false,
  "error": {
    "code": "SUBSCRIPTION_SOURCE_FAILED",
    "message": "订阅源 hysteria2 获取失败：连接超时",
    "details": {
      "source": "hysteria2",
      "reason": "ETIMEDOUT"
    }
  },
  "timestamp": "2026-06-24T12:00:00Z"
}
```

- **错误码体系**：

| 错误码 | HTTP 状态码 | 说明 |
|--------|------------|------|
| `UNAUTHORIZED` | 401 | 未提供有效凭证 |
| `FORBIDDEN` | 403 | 权限不足 |
| `NOT_FOUND` | 404 | 资源不存在 |
| `VALIDATION_ERROR` | 422 | 请求参数验证失败 |
| `SUBSCRIPTION_SOURCE_FAILED` | 502 | 上游订阅源获取失败 |
| `MIHOMO_UNAVAILABLE` | 503 | mihomo 服务不可用 |
| `CONFIG_INVALID` | 400 | 配置格式错误 |
| `INTERNAL_ERROR` | 500 | 内部错误 |

### 3. 请求验证

- **输入校验**：所有 API 端点使用 zod 校验输入参数
- **校验错误**：返回 422 + 明确的字段级错误信息
- **示例**：

```json
{
  "success": false,
  "error": {
    "code": "VALIDATION_ERROR",
    "message": "请求参数验证失败",
    "details": {
      "fields": [
        { "field": "content", "message": "订阅内容不能为空" },
        { "field": "content", "message": "订阅内容至少包含 20 个字符" }
      ]
    }
  }
}
```

### 4. API 版本控制

- **版本前缀**：`/api/v1/status`、`/api/v1/update` 等
- **向后兼容**：旧路径 `/api/status` 保留重定向到 `/api/v1/status`
- **废弃通知**：响应头中添加 `Deprecation: true` 和 `Sunset: <date>`

### 5. CORS 完善

- **可配置 CORS**：从 `config.yaml` 读取 `cors.origin`
- **预检请求**：正确处理 OPTIONS 请求
- **凭证支持**：支持 `Access-Control-Allow-Credentials`

## 验收标准

- [ ] `/api-docs` 页面可浏览完整的 OpenAPI 文档
- [ ] 所有 API 端点的错误响应格式统一
