# v0.2 — AI 项目记忆 & 开发者文档

**状态：待开发**

## 版本目标

让 AI（Claude Code）具备跨 session 的项目记忆能力，同时完善面向人类贡献者的文档体系，确保部署和 CI/CD 流程对 AI 和人类都清晰可查。

## 核心功能

### 1. CLAUDE.md 完整性增强

**现状**：124 行，涵盖架构、开发命令、部署、设计系统。
**目标**：补充以下章节，使 AI 无需额外探索即可理解项目全貌。

| 章节 | 内容 |
|------|------|
| 项目决策记录 | 关键架构决策及原因（为什么用 Next.js 而不用 Express？为什么用 mihomo 而非 subconverter？） |

| 数据流图 | 从 sing-box → 节点提取 → 转换 → Clash 配置的完整链路 |

### 2. AI 记忆系统

在 `.claude/memory/` 下建立持久化记忆文件，使 AI 在后续 session 中自动加载：

```
.claude/memory/
├── MEMORY.md              # 索引文件，一行一个记忆链接
├── project-architecture.md # 项目架构决策
├── deployment-flow.md      # 部署流程（systemd + nginx + GitHub Actions）
├── ci-cd-pipeline.md       # CI/CD 流水线说明
├── bug-fixes.md            # 历史 bug 及修复方案（避免重复踩坑）
├── config-patterns.md      # 配置约定和默认值
└── coding-conventions.md   # 代码风格和约定
```

**记忆内容示例**：

- `project-architecture.md`：为什么选 Next.js standalone、为什么 mihomo 只做本地命令行调用
- `deployment-flow.md`：`server-deploy.sh` 的原子部署流程、systemd 单元配置、Nginx 反代规则
- `ci-cd-pipeline.md`：GitHub Actions workflow 的触发条件、secrets 说明、部署步骤详解
- `bug-fixes.md`：yq `-o yaml` 缺失问题、api.ts 数据解包问题、版本号不一致问题

### 4. CI/CD 文档

**新文件**：`docs/CI-CD.md`
