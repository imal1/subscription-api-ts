# v1.2 — 在线配置管理

**状态：待开发**

## 版本目标

将配置从手动编辑 `config.yaml` 迁移到仪表盘在线管理，减少服务器 SSH 操作，降低配置错误率。

## 核心功能

### 1. 配置管理页面

新增 `/settings` 路由，提供可视化配置编辑：

| 配置区域 | 可编辑项 |
|----------|----------|
| 基础设置 | 服务端口、运行环境 |
| 协议配置 | sing-box 配置名称列表（增删改） |
| 定时更新 | cron 表达式（带预览下次执行时间） |
| 网络设置 | Nginx 端口、代理端口、请求超时 |
| 外部访问 | 主机地址 |
| 日志 | 日志级别 |

### 2. 配置持久化

- **读写流程**：仪表盘修改 → 调用 `/api/yaml/config` → 写入 `config.yaml` → 部分配置热生效（端口等需要重启）
- **配置验证**：提交前用 yq 验证 YAML 语法，用 mihomo 验证 Clash 配置
- **配置备份**：每次修改前自动备份到 `backup/` 目录
- **配置回滚**：提供配置历史查看和回滚功能

### 3. 配置热更新

支持无需重启的配置变更：

| 配置项 | 热更新 |
|--------|--------|
| sing-box 配置列表 | ✅ |
| cron 表达式 | ✅（重启 cron job） |
| 日志级别 | ✅ |
| 请求超时 | ✅ |
| 服务端口 | ❌（需重启） |
| Nginx 端口 | ❌（需重启） |

### 4. 配置导入导出

- **导出**：一键下载当前 `config.yaml`
- **导入**：上传 YAML 文件合并配置（不覆盖未上传的字段）
- **重置**：恢复默认配置

## 验收标准

- [ ] `/settings` 页面可正常访问，所有配置区域可编辑
- [ ] 修改配置后自动备份旧配置到 `backup/` 目录
- [ ] 配置验证通过后才写入文件
- [ ] 配置历史列表可查看，支持回滚到任意历史版本
- [ ] cron 表达式修改后定时任务立即生效（无需重启）
- [ ] 导入导出功能正常
- [ ] 配置错误时给出明确的中文错误提示
