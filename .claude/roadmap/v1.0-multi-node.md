# v1.0 — 多节点分布式管理系统

**状态：设计中**

## 版本目标

将 MioBridge 从单机订阅转换服务升级为分布式多节点管理系统。控制面部署在其中一台节点服务器上，通过公网直连 + HMAC 签名认证管理 2-10 台地域分散的节点。节点支持 sing-box、xray、v2ray 三种代理内核（均来自 233boy 仓库）。

详见设计文档：[2026-06-26-multi-node-distributed-system-design.md](../../docs/superpowers/specs/2026-06-26-multi-node-distributed-system-design.md)

## 核心功能

### 1. 节点注册表

- `~/.config/miobridge/nodes.yaml` 管理所有节点
- 支持节点 ID、名称、地址、内核类型、地域标签
- `fs.watch` 热加载，无需重启

### 2. NodeManager 服务

- 并发轮询远程节点状态（30s 间隔）
- 内存缓存聚合结果
- 支持单节点和批量操作

### 3. HMAC 签名认证

- 控制面：对远程请求进行 HMAC-SHA256 签名
- 远程节点：中间件验证签名 + 时间戳防重放（±30s 窗口）
- localhost 请求跳过验证

### 4. Dashboard 集群视图

- 多节点卡片总览（在线状态、内核类型、代理数、协议分布）
- 节点详情面板（展开单节点完整信息）
- 批量操作：全部更新、全部健康检查

### 5. 内核适配器

- `SingBoxAdapter` — 现有逻辑封装
- `XrayAdapter` — 解析 233boy xray 配置
- `V2rayAdapter` — 解析 233boy v2ray 配置

### 6. API 扩展

| 端点 | 用途 |
|------|------|
| `GET /api/cluster/status` | 聚合所有节点状态 |
| `POST /api/cluster/update` | 触发节点更新 |
| `GET /api/cluster/health` | 批量健康检查 |

## 验收标准

- [ ] `nodes.yaml` 配置的远程节点在 Dashboard 正确显示
- [ ] 控制面能通过 HMAC 签名调用远程节点 API
- [ ] 远程节点 HMAC 中间件正确验证签名并防重放
- [ ] Dashboard 显示所有节点的聚合状态
- [ ] 单节点详情面板展开显示完整信息
- [ ] 批量更新所有节点订阅
- [ ] 批量健康检查返回每个节点的延迟
- [ ] xray / v2ray 适配器正确提取节点 URL
- [ ] 单机部署（无 nodes.yaml）向后兼容，行为不变
- [ ] 远程节点离线时不影响控制面和其他节点
