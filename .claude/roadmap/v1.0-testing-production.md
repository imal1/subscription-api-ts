# v1.0 — 测试框架 & 生产就绪

**状态：待开发**

## 版本目标

建立完整的测试体系，达到生产级质量标准，使项目成为可靠的、可维护的开源订阅转换服务。

## 核心功能

### 1. 测试框架搭建

- **测试运行器**：Vitest（与 Vite/Next.js 生态兼容，速度快）
- **测试类型**：

| 类型 | 工具 | 覆盖目标 | 说明 |
|------|------|----------|------|
| 单元测试 | Vitest | ≥ 80% | 所有 service 层的纯函数 |
| 集成测试 | Vitest + Supertest | ≥ 60% | API 端点（mock 外部依赖） |
| E2E 测试 | Playwright | ≥ 5 条关键路径 | 仪表盘完整用户流程 |

### 2. 单元测试覆盖

**测试范围**：

- `mihomoService`：parseVless、parseVmess、parseTrojan、parseHysteria2、parseTuic、parseShadowsocks、generateClashConfig、convertObjectToYaml
- `mioBridgeService`：getStatus、ensureDirectories、createDefaultRawFile
- `singBoxService`：getAllConfigUrls（mock exec）
- `yamlService`：getConfigPath、validateConfig、getFullConfig

**最少测试用例数**：50+

### 3. 集成测试覆盖

**测试范围**：

- `GET /api/health` → 200 + 正确响应格式
- `GET /api/status` → 200 + data 字段完整
- `GET /api/update` → 200 + clashGenerated = true（mock mihomo/sing-box）
- `POST /api/convert` → 200 + 有效的 clash 配置（mock mihomo）
- `GET /subscription.txt` → 200 + base64 格式（mock 文件系统）
- `GET /clash.yaml` → 200 + 包含 proxies 字段（mock 文件系统）
- 错误场景：mihomo 不可用、sing-box 不可用、空订阅内容

**最少测试用例数**：20+

### 4. E2E 测试覆盖

**关键用户路径**：

1. **仪表盘加载**：访问 `/` → 看到状态卡片和节点数量
2. **订阅更新**：点击更新按钮 → 看到加载状态 → 更新完成显示结果
3. **在线转换**：打开转换弹窗 → 粘贴订阅内容 → 点击转换 → 看到 Clash 配置
4. **文件下载**：直接访问 `/clash.yaml` → 下载文件内容正确
5. **API 文档**：访问 `/api-docs` → 看到完整 API 文档

**最少测试用例数**：5+（覆盖 5 条关键路径）

### 5. 测试基础设施

- **CI 集成**：GitHub Actions 工作流中增加 `test` job
- **覆盖率报告**：使用 `@vitest/coverage-v8`，CI 中生成覆盖率报告
- **覆盖率门槛**：低于 70% 时 CI 失败（warning 模式，先不阻塞）
- **测试数据管理**：使用 fixtures 目录管理测试用数据（示例节点、示例配置）

### 6. CI/CD 完善

- **PR 检查**：lint + typecheck + test
- **合并前检查**：所有检查通过 + 覆盖率不下降
- **测试报告**：在 PR 评论中展示测试结果摘要

### 7. 项目元数据完善

- **README 完善**：添加测试徽章、覆盖率徽章、贡献指南链接
- **CHANGELOG**：从 v0.1 开始记录所有版本变更
- **License**：确认 MIT 许可证

## 验收标准

- [ ] `bun run test` 执行所有测试套件
- [ ] 单元测试 ≥ 50 个用例，覆盖率 ≥ 80%
- [ ] 集成测试 ≥ 20 个用例，覆盖率 ≥ 60%
- [ ] E2E 测试 ≥ 5 条关键路径全部通过
- [ ] GitHub Actions 包含 lint + typecheck + test job
- [ ] 测试报告在 CI 中自动生成
- [ ] README 包含测试和覆盖率徽章
- [ ] CHANGELOG.md 存在且包含 v0.1 到 v1.0 的完整变更记录
