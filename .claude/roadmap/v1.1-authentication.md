# v1.1 — API Key 鉴权 & 管理员登录

**状态：待开发**

## 版本目标

为服务添加基础鉴权：管理员通过用户名密码登录仪表盘，普通用户通过 API Key 访问订阅端点，未授权请求被拒绝。

## 核心功能

### 1. 管理员登录

- **登录页面**：`/login` 路由，用户名 + 密码表单
- **Session 管理**：使用 iron-session 或 jose 实现加密 cookie session
- **凭证配置**：管理员用户名和密码通过环境变量 `ADMIN_USERNAME` / `ADMIN_PASSWORD` 配置
- **登录保护**：仪表盘 `/` 和所有 `/api/*` 管理端点需要登录后才能访问
- **登出**：清除 session，跳转回登录页

### 2. API Key 鉴权

- **Key 生成**：管理员在仪表盘可以生成 / 吊销 API Key
- **Key 存储**：存储在 `~/.config/miobridge/api-keys.json`
- **端点保护**：`/subscription.txt`、`/clash.yaml`、`/raw.txt` 需要 API Key
- **传递方式**：支持两种方式
  - Query 参数：`/subscription.txt?key=<API_KEY>`
  - Header：`Authorization: Bearer <API_KEY>`
- **速率限制**：每个 API Key 每分钟最多 60 次请求

### 3. 中间件架构

```typescript
// 鉴权中间件
function authMiddleware(handler, options: { requireAdmin?: boolean }) {
  // 1. 检查 session（管理员）
  // 2. 检查 API Key（普通用户）
  // 3. 都不满足 → 401/403
}
```

**端点分类**：

| 端点类型 | 鉴权方式 | 说明 |
|----------|----------|------|
| `/login` | 无 | 登录页面 |
| `/` (仪表盘) | Session | 管理员登录后访问 |
| `/api/status` | Session | 管理 API |
| `/api/update` | Session | 管理 API |
| `/api/convert` | Session | 管理 API |
| `/api/configs` | Session | 管理 API |
| `/subscription.txt` | API Key | 订阅端点 |
| `/clash.yaml` | API Key | 订阅端点 |
| `/raw.txt` | API Key | 订阅端点 |
| `/api/health` | 无 | 健康检查保持公开 |

### 4. 仪表盘增强

- **API Key 管理面板**：在仪表盘中新增"访问控制"卡片
  - 查看所有 Key（名称、创建时间、最后使用时间、请求次数）
  - 创建新 Key（填写名称 → 生成 → 仅显示一次完整 Key）
  - 吊销 Key（确认对话框）
- **Key 命名**：每个 Key 有标签（如"Clash 客户端"、"Quantumult X"），便于识别

### 5. 安全措施

- **密码哈希**：管理员密码使用 bcrypt 或 argon2 哈希存储
- **Session 过期**：默认 24 小时，可配置
- **Key 格式**：`sub_` 前缀 + 32 字节随机 hex（如 `sub_a1b2c3d4...`）
- **暴力破解防护**：登录失败 5 次后锁定 15 分钟
- **HTTPS 强制**：生产环境建议通过 Nginx 配置 HTTPS

## 验收标准

- [ ] `/login` 页面可正常访问，登录成功后跳转仪表盘
- [ ] 未登录访问 `/` 被重定向到 `/login`
- [ ] 未提供 API Key 访问 `/subscription.txt` 返回 401
- [ ] 有效 API Key 访问 `/subscription.txt` 正常返回订阅内容
- [ ] 仪表盘可以创建、查看、吊销 API Key
- [ ] 登录失败 5 次后触发锁定
- [ ] Session 过期后自动跳转登录页
- [ ] `/api/health` 无需鉴权仍可访问（兼容部署健康检查）
- [ ] 环境变量 `ADMIN_USERNAME` / `ADMIN_PASSWORD` 未设置时服务启动失败并给出明确提示

## 依赖

- `iron-session` 或 `jose`（session 管理）
- `bcryptjs` 或 `@noble/hashes/argon2`（密码哈希）
