## 版本目标

交付一个功能完整、无已知缺陷的订阅转换服务，能够从 sing-box 获取节点、转换为 Clash 配置、提供 Web 仪表盘。

- [x] 服务可正常启动，`/api/health` 返回 200
- [x] 从 sing-box 获取节点 URL 正常
- [x] 订阅文件 `subscription.txt` 正常生成（base64 编码）
- [x] 原始节点文件 `raw.txt` 正常生成
- [x] Clash 配置文件 `clash.yaml` 正常生成（YAML 格式）

- [x] 定时更新（node-cron）正常执行

- [x] systemd 服务正常运行
- [x] Nginx 反代正常

## 技术栈

- **后端**：Next.js Pages Router + Node.js runtime（standalone 输出）
- **转换内核**：mihomo（clash-meta）
- **配置解析**：yq（mikefarah/yq v4）
- **前端**：React + Tailwind CSS + shadcn/ui（Botanical Garden 主题）
- **日志**：Winston
- **定时任务**：node-cron
- **部署**：systemd + Nginx + GitHub Actions

## 已修复的 Bug

1. **yq YAML 输出格式**：`yq eval -P` 输出 JSON 而非 YAML，导致 `clash.yaml` 永远无法生成。修复：添加 `-o yaml` 参数。
2. **仪表盘状态轮询丢失**：API 响应包含 `{ success, data }` 包装，但客户端未解包 `data` 字段，导致 30s 轮询后所有状态显示"未生成/0"。修复：`api.ts` 中解包 `response.data`。
3. **版本号不一致**：三处硬编码不同版本号。修复：统一从 `package.json` 读取。
4. **部署版本不可见**：无法确认服务器运行的 commit。修复：注入 `NEXT_PUBLIC_GIT_COMMIT`，仪表盘页脚显示。
