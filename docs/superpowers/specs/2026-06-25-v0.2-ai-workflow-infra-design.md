## 模块三：GitHub Actions 深度集成

### 3.1 `ci.yml` — PR 门禁（新建）

**触发条件**：PR 到 `main` 分支，或手动 `workflow_dispatch`

**Job 链路**：`lint → typecheck → build`

```
lint:
  - Checkout + Setup Bun + Install
  - bun run lint（oxlint）

typecheck (needs: lint):
  - Checkout + Setup Bun + Install
  - npx tsc --noEmit（TypeScript 类型检查）

build (needs: typecheck):
  - Checkout + Setup Bun + Setup Node
  - bun run build（next build）
  - 验证 standalone 产物存在（frontend/server.js）
```

**失败行为**：PR 显示 ❌，阻止 merge。

### 3.2 `deploy.yml` — 增强（在已有基础上）

**保留现有全部步骤**，新增/修改：

1. **注入 commit 信息**：构建环境增加 `NEXT_PUBLIC_GIT_COMMIT=${{ github.sha }}`
2. **Step Summary 增强**：部署成功时显示 release id、commit link、部署耗时
3. **失败时输出排查命令**：`journalctl -u miobridge -n 50`、`readlink ~/.config/miobridge/dist`

### 3.3 `health-check.yml` — 定时健康检查（新建）

**触发条件**：`schedule: "*/5 * * * *"` + `workflow_dispatch`

```
job: health-check
  steps:
    1. SSH 到服务器 curl /api/health
    2. 成功 → Step Summary 记录 ✅
    3. 失败 → systemctl restart miobridge → sleep 10 → curl /api/health
       - 恢复 → Step Summary 记录 ⚠️ 已自动恢复
       - 仍失败 → Step Summary 记录 ❌ 需人工介入
```

**安全考虑**：SSH 使用已有的 `DEPLOY_SSH_KEY` secrets。重启使用 `sudo systemctl restart`（已有 NOPASSWD sudoers 配置）。

### 3.4 CI/CD 全流程总览

```
PR opened ──→ ci.yml ──→ lint → typecheck → build ──→ ✅ merge allowed
                                                          │
push to main ──→ deploy.yml ──→ build → pack → deploy → health check ──→ ✅
                                                          │
cron */5 ──→ health-check.yml ──→ ping → 正常/自动恢复/告警
```

## 模块四：Claude Code 整合层

### 4.1 `.claude/settings.json`

```json
{
  "permissions": {
    "allow": [
      "Bash(bun:*)",
      "Bash(gh:*)",
      "Bash(systemctl:status miobridge)",
      "Bash(journalctl:-u miobridge*)",
      "Bash(curl:localhost:*)",
      "Bash(curl:/api/health*)",
      "Bash(git:*)",
      "Bash(oxlint:*)",
      "Bash(npx:tsc*)",
      "Bash(readlink:*)",
      "Bash(ls:~/.config/miobridge*)",
      "Bash(cat:~/.config/miobridge*)"
    ]
  },
  "hooks": {
    "PreCommit": [
      {
        "command": "bun run lint"
      },
      {
        "command": "bash -c 'MSG=$(git rev-parse --git-dir)/COMMIT_EDITMSG; if [ -f \"$MSG\" ]; then head -1 \"$MSG\" | grep -qE \"^(feat|fix|docs|refactor|chore|test|ci|perf|style|build):\" || echo \"⚠️  Commit 消息不符合 Conventional Commits 格式\"; fi'"
      }
    ]
  }
}
```

### 4.2 自定义 Slash Commands

#### `/miobridge:deploy` — 一键部署

1. 确认当前分支和未推送 commits
2. `git push origin <branch>`
3. 若 push 到 main → 自动触发 deploy.yml
4. `gh run watch` 等待 CI 完成
5. `curl /api/health` 二次确认
6. 报告结果（release id、commit、耗时）

#### `/miobridge:diag` — 诊断

1. `curl /api/health` — 服务状态
2. `curl /api/status` — 详细状态
3. `gh run list -w deploy.yml -L5` — 最近部署记录
4. `gh run list -w health-check.yml -L3` — 最近健康检查
5. `journalctl -u miobridge --since "10 min ago"` — 最近日志
6. `readlink ~/.config/miobridge/dist` — 当前运行版本
7. 汇总报告

#### `/miobridge:release` — 发布

1. 从 CHANGELOG.md 读取待发布版本
2. 确认版本号 → `git tag vX.Y.Z`
3. `git push --tags` → 触发 deploy.yml
4. 等待部署完成 → 报告
