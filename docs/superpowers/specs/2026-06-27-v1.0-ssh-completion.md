## 一、DeployManager SSH 实战

### 新增方法

| 方法 | 说明 |
|------|------|
| `testConnection(nodeId)` | SSH 连接测试，返回 OS/架构/Bun 版本 |
| `fullDeploy(nodeId)` | AsyncGenerator，6 步全流程（连接→Bun→内核→Agent→启动→验证） |
| `updateAgent(nodeId)` | 从 GitHub Releases 下载最新二进制，替换后重启 |
| `uninstallAgent(nodeId)` | systemctl stop + disable + 删除二进制/配置/systemd unit |
| `restartAgent(nodeId)` | systemctl restart miobridge-agent |
| `stopAgent(nodeId)` | systemctl stop miobridge-agent |
| `startAgent(nodeId)` | systemctl start miobridge-agent |
| `getLatestReleaseVersion()` | 查询 GitHub Releases 最新 tag |
| `checkAgentUpdates()` | 对比所有节点 agent 版本 vs 最新 release |

### 部署流程（AsyncGenerator + SSE）

```
fullDeploy(nodeId):
  step 1: testConnection → OS/arch 检测 → yield {step:'connect', progress:10}
  step 2: installBun → curl bun.sh/install → yield {step:'bun', progress:25}
  step 3: installKernel → 233boy 脚本 → yield {step:'kernel', progress:50}
  step 4: deployAgent → scp/curl 二进制 + agent.yaml + systemd → yield {step:'agent', progress:75}
  step 5: startService → systemctl enable --now → yield {step:'start', progress:90}
  step 6: healthVerify → curl /api/health → yield {step:'verify', progress:100}
```

SSE 端点 `GET /api/cluster/deploy/progress?node=id` 消费 AsyncGenerator，实时推送。

### Agent 二进制来源

- **首次部署**：从 GitHub Releases 下载对应架构二进制（`miobridge-agent-linux-amd64` / `-arm64`）
- **后续更新**：NodeManager 轮询时对比 `node.agent.version` vs 最新 release tag
- **版本检测**：`GET /api/cluster/releases/latest` 返回最新 tag 和下载 URL
- **降级**：如果 GitHub 不可达，支持手动指定下载 URL

### SSH 认证双通道

方式1 — **上传私钥文件**：拖拽 .pem/.key 文件，保存到 `~/.config/miobridge/ssh/<node-id>.key`（600）

方式2 — **ssh-copy-id**：用户提供用户名 + 密码（一次性），控制面 `ssh-keygen -t ed25519` 生成密钥对 → 密码 SSH 登录 → 写入 authorized_keys → 密码丢弃

## 二、nodes.yaml 状态机

### 状态定义

```
not_deployed ──部署──▶ deploying ──成功──▶ running ──停止──▶ stopped
                         │                    │  ▲              │
                         │失败                │  │启动          │
                         ▼                    │  └──────────────┘
                       error                  │
                         │                    │更新
                         │重新部署            ▼
                         └────────────────▶ running ◀── 更新成功 ── updating
                                              │                      ▲
                                              │停止                  │
                                              ▼                      │
                                           stopped ──── 更新 ────────┘

卸载流程:
  running/stopped/error ──卸载──▶ uninstalling ──完成──▶ not_deployed
```

### 允许的操作（状态守卫）

| 操作 | 允许的状态 |
|------|-----------|
| 部署 | `not_deployed`, `error` |
| 停止 | `running` |
| 启动 | `stopped` |
| 更新 | `running`, `stopped` |
| 重启 | `running` |
| 卸载 | `running`, `stopped`, `error` |
| 重新部署 | `error` |

### nodes.yaml 新增字段

```yaml
nodes:
  - id: "node-sg"
    agent:
      deployed: false
      version: ""
      status: "not_deployed"     # 状态机当前状态
      statusHistory:             # 最近 10 条状态变更
        - from: "not_deployed"
          to: "deploying"
          at: "2026-06-27T14:00:00Z"
          reason: "用户触发部署"
      lastDeploy: ""
      lastError: ""
```

### nodes.yaml 读写 API

| 端点 | 方法 | 说明 |
|------|------|------|
| `POST /api/cluster/nodes` | POST | 添加/更新节点（序列化写入 nodes.yaml） |
| `DELETE /api/cluster/nodes?node=id` | DELETE | 删除节点（从 nodes.yaml 移除） |
| `GET /api/cluster/releases/latest` | GET | 查询 GitHub 最新 release 版本 |

NodeManager 已有 `fs.watch` 热加载，写入后自动生效。

## 四、任务分解

### Task 1: DeployManager SSH 连接管理
- `connect(nodeId)` — 读取 SSH 配置，建立 node-ssh 连接
- `testConnection(nodeId)` — 连接测试 + OS/arch/Bun 检测
- `disconnect(nodeId)` — 断开连接
- SSH 连接池管理

### Task 2: DeployManager fullDeploy 全流程
- 6 步 AsyncGenerator 实现
- Bun 安装（curl bun.sh/install）
- 内核安装（233boy 脚本）
- Agent 部署（下载二进制 + scp agent.yaml + systemd unit）
- 启动 + 健康验证

### Task 3: Agent 更新/生命周期管理
- `updateAgent` — GitHub Releases 下载 + 替换 + 重启
- `restartAgent/stopAgent/startAgent` — systemctl 操作
- `uninstallAgent` — 清理二进制/配置/systemd unit
- `getLatestReleaseVersion` + `checkAgentUpdates`

### Task 4: nodes.yaml 状态机 + 读写
- 状态机实现（状态守卫、状态转换、statusHistory）
- `POST /api/cluster/nodes` 写入端点
- `DELETE /api/cluster/nodes` 删除端点
- NodeManager 扩展：updateNodeStatus、writeNodesYaml

### Task 5: AddNodeForm SSH 双通道
- 文件上传（拖拽 .pem/.key）
- ssh-copy-id（密码输入 + 自动生成密钥对）
- 表单验证

### Task 6: SSE 部署进度实时推送
- DeployManager.fullDeploy() AsyncGenerator 接 SSE
- Dashboard DeployProgressDialog 实时更新

### Task 7: 文档更新
- README.md / PROJECT_STRUCTURE.md / CHANGELOG.md / CLAUDE.md / TROUBLESHOOTING.md
